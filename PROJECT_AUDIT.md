## 8. Potential Conflicts & Issues

### Critical (blocks functionality)

1. **DiscoveryService.\_deviceKey() uses metadata fields never populated by mDNS** — deduplication always falls through to IP, which means duplicate devices from multiple providers may show.
2. **No timeout on mDNS discovery** — `_client!.lookup<>()` in MdnsDiscoveryProvider waits indefinitely if no devices respond. No cancellation mechanism.
3. **No refresh/discovery restart** — once started, discovery runs once and stops. No way to re-scan without re-creating the service.
4. **TXT record parsing is correct** for multicast_dns v0.3.3 (uses `writeln` → `\n` separator). Split logic works.
5. **DiscoveryBloc** needs to be examined — events/states may need wiring to datasource.

### Minor

6. **4 `use_super_parameters` lint infos** in DriverException constructors (cosmetic, Dart 3.11 feature).
7. **Settings feature** has only domain/repository interface, no implementation or UI.
8. **Shared widgets** directory empty — no reusable components.

## 9. Risks

- Hive CE adapter must be registered on all platforms
- Android TV pairing requires certificate management (already implemented in core/drivers/android_tv/pairing/)
- Android TV remote uses protobuf over TLS (already implemented in protocol/)
