# Project Context

## Overview

Flutter AI Workspace (FAW) is a reusable engineering knowledge base for AI-augmented Flutter development. It standardizes architecture, tooling, and workflows across projects, enabling consistent, production-quality output from both human developers and AI agents.

## Scope

- **In scope:** Flutter/Dart mobile apps using Clean Architecture, Bloc, Firebase, and the full FAW tech stack.
- **Out of scope:** Game development, AR/VR, desktop-only apps, non-Flutter projects.

## Key Stakeholders

- **AI Agents:** Cline, Claude Code, Copilot, Roo Code—primary consumers of FAW documentation.
- **Flutter Developers:** Human engineers writing code, reviewing PRs, maintaining projects.
- **Architects:** Engineers defining system design, making technical decisions.
- **Project Managers:** Using checklists and playbooks for process tracking.

## Core Goals

1. **AI-First:** Documentation structured for AI parsing and execution.
2. **Documentation-First:** Standards documented before code is written.
3. **Planning Before Coding:** Every task requires a defined plan first.
4. **Production Ready:** All patterns target real-world, shipped applications.
5. **Maintainable:** Minimal tech debt through enforced standards.
6. **Reusable:** Across projects, not tied to any single app.

## Success Criteria

- A new Flutter project can be bootstrapped with full architecture in < 30 minutes.
- AI agents implement features correct to standards on first pass.
- Code review checklist catches > 90% of standard violations automatically.
- New developer onboarding time < 1 day for FAW-conformant projects.
